# 安全政策

## 支持的版本

| 版本 | 支持状态 |
| ------- | ------------------ |
| 1.0.x   | ✅ 积极支持 |
| < 1.0   | ❌ 不支持 |

## 报告漏洞

我们非常重视安全问题。如果你发现安全漏洞，请负责任地披露。

### 📧 报告方式

**请不要**通过公开 Issue 报告安全漏洞。

请发送邮件至：**security@nvwax.dev**

邮件内容应包括：
- 漏洞类型
- 完整描述
- 重现步骤
- 潜在影响
- 建议的修复方案（如有）

### 🔒 保密协议

我们承诺：
- 在 48 小时内确认收到报告
- 在 7 天内提供初步评估
- 在修复前保密漏洞信息
- 修复后公开致谢（如你同意）

## 安全最佳实践

### 对于用户

1. **保持更新**: 使用最新版本
2. **强密码**: 使用复杂密码并定期更换
3. **双因素认证**: 启用 2FA（计划中）
4. **警惕钓鱼**: 不要泄露凭证

### 对于开发者

1. **依赖检查**: 定期审计依赖包
2. **代码审查**: 所有 PR 必须经过审查
3. **最小权限**: 遵循最小权限原则
4. **输入验证**: 验证所有用户输入
