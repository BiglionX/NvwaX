## 🔄 维护说明

### 何时归档文档

以下类型的文档应该归档：
1. 阶段性完成报告（COMPLETION, REPORT）
2. 临时测试文档（TEST, GUIDE）
3. 过时的设计草案（PLAN, DESIGN）
4. 已被整合的功能文档

### 何时保留文档

以下类型的文档应该保留：
1. 核心用户文档（README, GUIDE）
2. API 参考文档
3. 部署和维护指南
4. 最新的项目总结和进展

### 归档流程

1. 识别可归档的文档
2. 移动到 `docs-archive/` 对应子目录
3. 更新本索引文件
4. 提交更改
