为了让开发者最高效地把我们的能力接入他们的网站，我们可以提供一套组合拳。针对不同开发者的技术栈、安全要求和产品定位，有四种主流的集成方案可供选择：

### 🚀 主流集成方式对比

| 模式 | 技术门槛 | 功能完整度 | 集成速度 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **iframe** | 极低 | 基础 | 最快，几分钟实现 | 快速部署标准对话窗口 |
| **JS SDK** | 低 | 较高（可定制主题、交互） | 快 | 需轻度定制外观和行为的场景 |
| **API 集成** | 中/高 | 极高（完全可编程） | 取决于开发 | 需深度功能整合或复杂流程 |
| **微前端 (Web Components)** | | | 中 | 希望UI组件与主站样式严格隔离，但又需深度嵌入 |

### 核心产品集成

#### 1. 营销 AI Team：智能销售转化机器人
将该技能整体打包为一个可直接调用的 **API** 或 **SDK**，只需几段代码即可将 AI 营销团队的能力植入网站后台，实现**用户意图自动识别**和**24小时智能问答**。
- **API 集成步骤**：
    1. 在管理后台生成一个 **开发者凭证**（如 `api_key: YOUR_API_KEY`）。
    2. 开发者在后端通过 HTTP 请求 `/v1/chat/completions` 发送用户问题（`messages` 参数）和自己的凭证（`api_key`）。
    3. NvWaX 平台接收请求，调用 AI 分析并返回结果。
    4. 开发者将结果在前端展示给用户。

#### 2. 智能体广场：创建免费组件吸引用户
将智能体广场封装为一个独立的 **Web Component**，类似于一个功能完整的“智能体商店”，让任何网站都能一键嵌入。
- **Web Component 集成方式**：
    ```html
    <!-- 1. 引入封装好的自定义元素脚本 -->
    <script src="https://cdn.nvwax.com/agent-marketplace.js"></script>
    
    <!-- 2. 在网页任意位置，像使用原生标签一样使用它 -->
    <nvwax-marketplace site-id="YOUR_SITE_ID" theme="light"></nvwax-marketplace>
    ```
    该组件会自动加载并渲染整个智能体广场界面，从我们的平台获取数据，提供浏览、搜索和展示功能。

#### 3. 用户自定义智能体：提供自助式AI开发平台
将其打造成一个包含强大 **AI 编排引擎**（**Agent Hub**）的**微前端应用**，开发者可以通过一个 `iframe` 或 `npm` 包的形式，将一套功能完备的 **AI 智能体可视化开发工具**完整嵌入到自己的网站上。
- **iframe 方式**：将 Agent Hub 作为一个独立服务部署，通过 iframe 嵌入，并利用 **PostMessage API**（允许来自不同源的页面之间进行安全通信） 监听其产生的所有创建和保存事件，实现与主站的数据交互。
- **npm 包方式**：提供一个标准的 **npm 包**（如 `@nvwax/agent-studio`），开发者可以将其集成到自己的 React/Vue 项目中，调用其提供的 React 组件和 API，获得最高的定制自由度。

### ⚙️ 关键问题设计

#### 1. 安全与租户隔离
这是企业级应用的基础，需采用**多层隔离架构**：
- **API 层**：分发唯一的 `api_key`，每次请求校验其有效性并关联到特定开发者/团队。
- **数据库层**：为每个租户的数据（如配置、记忆、向量）添加 `tenant_id` 字段进行硬隔离，并在所有查询语句中加入该字段进行过滤，这是防止数据泄露最根本的手段。
- **运行时层**：使用 **OpenHermit**等工具或容器化技术（如 Docker）将不同租户的 Agent 运行时环境隔离开，防止恶意代码或资源滥用影响他人。
- **权限层**：引入 **RBAC** (Role-Based Access Control) 模型，在租户内部对成员进行精细的权限控制。

#### 2. 技能与工具扩展
通过 **统一的工具（Tool）标准** 扩展智能体能力：
- **定义标准结构**：每个工具都遵循 `name`（名称）、`description`（能给 AI 提供明确的功能描述）、`input_schema`（使用 JSON Schema 定义预期的参数）和 `execute`（具体实现代码）的通用结构，便于 AI 识别和调用。
- **注册与发现**：定义一个标准的工具注册中心（Tool Registry），开发者只需遵循 `Tool` 接口规范，即可轻松将外部 API、脚本或本地命令注册为智能体的工具。

#### 3. 数据主权与隐私合规
严格遵守全球数据隐私法规（如 GDPR）：
- **明确目标**：不同场景下对数据隐私的要求不同，例如外贸网站涉及商业机密和客户隐私，严格程度高于普通引流网站。
- **数据本地化**：提供私有化部署方案，允许开发者将整个 NvWaX 平台部署在自己的服务器或 VPC 中，所有数据完全由开发者掌控，适用于外贸等敏感场景。
- **透明与授权**：在 SDK 或 API 调用时，明确告知用户哪些数据将被收集及用途，并获得其明示同意。为用户或开发者提供查看、导出和删除其数据的入口。在传输和存储时，对个人身份信息（PII）进行匿名化或假名化处理。在服务条款中清晰界定数据的所有权和使用权归属。

具体到你的三个场景，建议如下集成方案：

1.  **外贸网站 - 植入营销 AI Team**：作为一项高级功能，以外挂服务的形式提供；首选 **API 集成**，将 AI 营销能力无缝嵌入开发者的后台和业务流。
2.  **普通引流网站 - 增加智能体广场**：作为一个吸引用户和增加粘性的免费功能；首选 **Web Component**，为开发者提供最简单的“零成本”接入方式。
3.  **允许用户自定义智能体**：作为一个高级增值功能模块；根据开发者需求，提供 **iframe** 或构建一个标准的 **npm 包**，以提供最强大的自定义开发能力。
