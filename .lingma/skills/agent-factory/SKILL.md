# Agent Factory Skill - 智能体工厂

**重要**: 本 Skill 基于现有的 `skillhub-workflow` 引擎，不重复实现工作流逻辑。

## 📚 核心依赖

本项目使用 `packages/skillhub-workflow/` 中的现有模块：

- **Agent Orchestrator**: `src/agents/orchestrator.js` - 多 Agent 协调器
- **Agent Templates**: `src/workflows/agent-templates.js` - 工作流模板
- **Workflow Engine**: `src/server.js` - 工作流执行引擎

## 🎯 功能定位

本 Skill 的主要职责是：
1. **需求采集** - 通过对话引导用户明确智能体需求
2. **模板推荐** - 调用 skillhub-workflow 的模板搜索 API
3. **技能分析** - 分析用户需求与模板技能的匹配度（待实现）
4. **悬赏管理** - 发布和管理技能悬赏任务（待实现）
5. **工作流委托** - 将实际的智能体生成委托给 skillhub-workflow

## 🔄 工作流程

### Step 1: 需求采集

通过 4-5 个结构化问题收集用户需求：
1. 用途描述
2. 数据源
3. 输出类型
4. 实现方式（可选）
5. 特殊要求（可选）

### Step 2: 模板搜索

调用 skillhub-workflow API：
```bash
GET http://localhost:3002/api/workflows/templates?q={query}
```

### Step 3: 用户选择

展示 Top 3-5 个匹配模板供用户选择。

### Step 4: 技能分析（待实现）

对比用户需求与模板能力，识别技能缺口。

### Step 5: 处理缺失技能

- 推荐补充技能
- 或发布悬赏任务

### Step 6: 生成智能体

调用 skillhub-workflow API 创建工作流并执行：
```bash
POST http://localhost:3002/api/workflows
POST http://localhost:3002/api/workflows/:id/execute
```

## 📋 工作流程

### Step 1: 需求采集（多轮对话）

通过 4-5 个结构化问题引导用户明确需求：

#### 问题 1: 用途描述
```
您需要什么样的智能体？请描述它的主要用途。

例如：
- "需要一个能自动回复客户咨询的客服智能体"
- "想要一个能分析销售数据并生成报表的分析助手"
- "需要一个能帮我写代码的编程助手"
```

**等待用户回答** → 保存到 `user_requirement`

#### 问题 2: 数据源
```
这个智能体需要访问哪些数据或系统？

例如：
- "订单数据库、商品库存 API"
- "公司知识库、产品文档"
- "GitHub 仓库、Stack Overflow"
- "不需要访问外部数据"
```

**等待用户回答** → 保存到 `data_sources`

#### 问题 3: 输出结果
```
您希望它输出什么结果？

例如：
- "回复客户消息（文本）"
- "生成 Excel 报表和图表"
- "提供代码片段和解释"
- "发送电子邮件通知"
```

**等待用户回答** → 保存到 `output_types`

#### 问题 4: 实现方式（可选）
```
您希望它如何得到这个结果？（可选，可跳过）

例如：
- "调用现有的订单查询 API"
- "分析用户评论的情感倾向"
- "从知识库中检索相关信息"
- "使用 GPT-4 生成回复"
```

**等待用户回答** → 保存到 `implementation_method`（可为空）

#### 问题 5: 特殊要求（可选）
```
还有其他特殊要求吗？（可选，可跳过）

例如：
- "需要支持中文和英文"
- "响应时间要在 2 秒以内"
- "需要集成到微信企业号"
- "没有特殊要求"
```

**等待用户回答** → 保存到 `special_requirements`（可为空）

---

### Step 2: 模板搜索

调用后端 API 搜索匹配的模板：

```typescript
// 伪代码
const searchQuery = `${user_requirement} ${data_sources} ${output_types}`;
const templates = await fetch(`/api/templates/search?q=${encodeURIComponent(searchQuery)}&limit=5`);
```

**如果找到模板**:
- 展示 Top 3-5 个匹配模板
- 显示每个模板的：
  - 名称和描述
  - 包含的技能列表
  - 适用场景
  - 匹配度百分比
  - 星级和下载量

**如果没有找到模板**:
- 提示："未找到完全匹配的模板，但我可以帮您从零开始创建"
- 进入 Step 4（技能分析）

---

### Step 3: 用户选择模板

展示模板卡片，让用户选择：

```
找到了 X 个匹配的模板：

┌─────────────────────────────────────┐
│ 🤖 电商客服智能体                    │
│ 匹配度: 95% | ⭐ 1523 | 📥 3421    │
│                                     │
│ 包含技能:                           │
│ ✅ 客户服务                          │

│                                     │

└─────────────────────────────────────┘

请选择一个模板（输入编号），或输入"跳过"从零开始创建。
```

**用户选择** → 保存到 `selected_template_id`

### Step 4: 技能缺口分析

调用后端 API 分析技能缺口：

```typescript
// 伪代码
const analysis = await fetch('/api/skills/analyze', {
  method: 'POST',
  body: JSON.stringify({
    userRequirement: user_requirement,
    templateId: selected_template_id // 如果选择了模板
  })
});
```

**展示分析结果**:

```

   - 客户服务
   - 意图识别

   - 订单查询（匹配度 95%）

💡 建议:
   该模板已具备 80% 所需能力，只需补充"订单查询"技能即可满足您的需求。
```

**用户确认** → 询问是否继续

### Step 5: 处理缺失技能

**情况 A: 所有缺失技能都可在技能商店找到**

```
✅ 所有缺失技能都可以在技能商店中找到：

1. 订单查询技能
   - 评分: ⭐ 4.7
   - 下载量: 1523
   - 作者: 李四
   
是否自动添加这些技能并生成智能体？（是/否）
```

**情况 B: 部分技能不存在，需要发布悬赏**

```

1. 微信支付集成

B. 手动上传自己的技能包

```

**如果选择 A**:
- 调用 `/api/bounties` 创建悬赏
- 显示悬赏详情和进度追踪链接

## 🔧 技术实现

### 依赖服务

- **后端 API**: `http://localhost:3001/api`
- **模板搜索**: `GET /api/templates/search`
- **技能分析**: `POST /api/skills/analyze`
- **悬赏管理**: `POST /api/bounties`
- **工作流引擎**: skillhub-workflow

### 数据结构

```typescript
interface AgentFactoryContext {
  // 用户需求
  user_requirement: string;
  data_sources: string[];
  output_types: string[];
  implementation_method?: string;
  special_requirements?: string;
  
  // 模板选择
  selected_template_id?: string;
  selected_template_skills?: string[];
  
  // 技能分析
  required_skills: string[];
  available_skills: string[];
  missing_skills: string[];
  
  // 悬赏
  created_bounties?: Array<{
    id: string;
    title: string;
    rewardAmount: number;
  }>;
  
  // 生成的智能体
  generated_agent?: {
    id: string;
    url: string;
    apiKey: string;
  };
}
```

## 💡 最佳实践

### 1. 引导话术优化

- **友好自然**: 避免机械式提问，使用 conversational tone
- **提供示例**: 每个问题都给出 2-3 个具体示例
- **允许跳过**: 非必填问题明确标注"可选"
- **即时反馈**: 用户回答后给予确认（"好的，已记录"）

### 2. 错误处理

- **API 失败**: "抱歉，模板搜索暂时不可用，请稍后重试"
- **无匹配结果**: "未找到完全匹配的模板，但我们可以从零开始创建"
- **积分不足**: "您的积分余额不足，请先充值或完成每日签到"

### 3. 性能优化

- **缓存搜索结果**: 相同查询缓存 5 分钟
- **异步加载**: 先展示模板列表，再异步加载详情
- **超时处理**: API 调用超时 10 秒，超时后降级为本地搜索
