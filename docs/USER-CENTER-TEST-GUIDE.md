> **测试版本**: Agent Repository Refactor v1.0

## 📋 测试前准备

### 1. 启动服务
```bash
# 终端 1: 后端服务
cd packages/nvwax-server
npm run dev

# 终端 2: 前端服务
cd packages/nvwax-web
npm run dev
```

### 2. 访问地址
- 前端: http://localhost:3000
- 后端 API: http://localhost:3001

### 3. 登录账号
使用已有的测试账号登录,或注册新账号。

### ✅ Phase 2: Agent 仓库核心功能测试

访问路径: **用户中心 → 我的 Agent 仓库** (`/agent-repository`)

#### 2.1 页面加载
- [ ] 页面正常加载,无错误
- [ ] 顶部显示"我的Agent仓库"标题
- [ ] Tab 切换按钮显示: "Agents" 和 "AiTeams"
- [ ] 默认选中 "Agents" Tab
- [ ] 显示创建资源按钮

#### 2.2 创建 Agent
1. 点击"创建新资源"按钮
2. 在模态框中选择"Agent"类型
3. 输入名称(必填)
4. 输入描述(可选)
5. 点击"创建"按钮

**预期结果**:
- [ ] 创建成功,模态框关闭
- [ ] 列表中显示新创建的 Agent
- [ ] 显示状态为"私有"或"草稿"
- [ ] 版本号默认为 "1.0.0"

#### 2.3 创建 AiTeam
1. 点击"创建新资源"按钮
2. 选择"AiTeam"类型
3. 输入名称和描述
4. 点击"创建"

**预期结果**:
- [ ] 创建成功
- [ ] 切换到 "AiTeams" Tab 可以看到
- [ ] 成员数显示为 0

#### 2.4 Tab 切换
- [ ] 点击 "Agents" Tab,显示 Agent 列表
- [ ] 点击 "AiTeams" Tab,显示 AiTeam 列表
- [ ] 切换流畅,无闪烁
- [ ] 计数正确更新

### ✅ Phase 3: CRUD 功能详细测试

#### 3.1 查看详情 (View)
**测试步骤**:
1. 在 Agent/AiTeam 卡片上点击 👁️ Eye 图标

**预期结果**:
- [ ] 打开详情模态框
- [ ] 显示完整信息:
  - 基本信息(名称、版本、状态、类型)
  - 描述内容
  - 标签列表
  - 统计数据(下载、评分、评价)
  - 时间信息(创建时间、更新时间)
- [ ] AiTeam 额外显示:
  - 执行次数
  - 成功率
  - 团队成员列表
- [ ] 点击"关闭"按钮或背景遮罩关闭模态框

#### 3.2 编辑资源 (Edit)
**测试步骤**:
1. 点击 ✏️ Edit 图标
2. 修改名称
3. 修改描述
4. 添加标签(输入后按回车)
5. 删除标签(点击 ×)
6. 修改版本号
7. 修改分类
8. 点击"保存更改"

**预期结果**:
- [ ] 表单预填充当前数据
- [ ] 可以正常编辑所有字段
- [ ] 标签添加/删除功能正常
- [ ] 保存成功后模态框关闭
- [ ] 列表中显示更新后的数据
- [ ] 名称为必填,空值时保存按钮禁用

#### 3.3 删除资源 (Delete)
**测试步骤**:
1. 点击 🗑️ Trash2 图标
2. 确认对话框显示资源名称
3. 点击"取消" → 应该关闭对话框
4. 再次点击删除图标
5. 点击"删除"确认

**预期结果**:
- [ ] 显示确认对话框
- [ ] 消息中包含资源名称
- [ ] 取消操作不删除资源
- [ ] 确认后资源从列表中消失
- [ ] 数据库中的记录被删除

#### 3.4 导出资源 (Export)
**测试步骤**:
1. 点击 ⬇️ Download 图标
2. 选择导出格式:
   - JSON
   - YAML
   - ProClaw
3. 点击"开始导出"

**预期结果**:
- [ ] 导出模态框打开
- [ ] 三种格式选项清晰显示
- [ ] 选择后有视觉反馈(高亮)
- [ ] ProClaw 格式显示特别提示
- [ ] 导出过程显示加载动画
- [ ] 导出成功显示"导出成功"
- [ ] 1.5秒后自动关闭模态框
- [ ] 文件下载到本地(检查下载文件夹)

**验证导出文件**:
- [ ] JSON 格式: 可以用文本编辑器打开,格式正确
- [ ] YAML 格式: 人类可读,结构清晰
- [ ] ProClaw 格式: 包含 `proclaw_version` 字段

#### 3.5 发布/取消发布 (Publish)
**测试步骤**:
1. 找到状态为"私有"或"草稿"的资源
2. 点击 📤 Send 图标(紫色,提示"发布")
3. 等待操作完成

**预期结果**:
- [ ] 状态从"私有"变为"已发布"
- [ ] 徽章颜色从灰色变为绿色
- [ ] Send 图标变为黄色,提示"取消发布"
- [ ] 再次点击可以取消发布
- [ ] 取消发布后状态变回"私有"

### ✅ Phase 5: 边界情况测试

#### 5.1 表单验证
- [ ] 创建时名称为空 → 应该阻止提交
- [ ] 编辑时名称为空 → 保存按钮禁用
- [ ] 标签重复添加 → 应该去重
- [ ] 超长文本输入 → 应该正常显示(截断或换行)

#### 5.2 并发操作
- [ ] 快速连续点击删除 → 只删除一次
- [ ] 编辑时同时刷新页面 → 数据一致性
- [ ] 多个 Tab 同时操作 → 数据同步

#### 5.3 网络异常
- [ ] 断开网络后操作 → 显示错误提示
- [ ] 恢复网络后重试 → 操作成功

#### 5.4 大数据量
**测试场景**: 创建 50+ 个 Agent
- [ ] 列表滚动流畅
- [ ] 加载时间合理(<2秒)
- [ ] 内存占用正常

### ✅ Phase 6: API 集成测试

使用浏览器开发者工具(F12)检查 Network 面板:

#### 6.1 创建资源
- [ ] POST `/api/agents` 或 `/api/aiteams`
- [ ] 请求体包含正确的字段
- [ ] 响应状态码 201 或 200
- [ ] 返回完整的资源对象

#### 6.2 获取列表
- [ ] GET `/api/agents` 或 `/api/aiteams`
- [ ] 返回数组和总数
- [ ] 分页参数正确(如果有)

#### 6.3 更新资源
- [ ] PUT `/api/agents/:id` 或 `/api/aiteams/:id`
- [ ] 只发送修改的字段
- [ ] 响应包含更新后的数据

#### 6.4 删除资源
- [ ] DELETE `/api/agents/:id` 或 `/api/aiteams/:id`
- [ ] 响应状态码 200 或 204

#### 6.5 导出资源
- [ ] POST `/api/agents/:id/export` 或 `/api/aiteams/:id/export`
- [ ] 请求体包含 `format` 字段
- [ ] 响应包含下载链接或文件数据

#### 6.6 发布/取消发布
- [ ] POST `/api/agents/:id/publish` 或 `/unpublish`
- [ ] 响应包含更新后的 `publishStatus`
